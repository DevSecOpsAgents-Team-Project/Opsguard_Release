# 브랜치 병합 비교 및 선택 내용 문서

## 개요

`feature/dev`와 `feature/level-router` 브랜치를 비교하여 최적의 코드를 선택하고 병합한 내용을 정리합니다.

---

## 1. 브랜치별 주요 특징

### feature/dev 브랜치
- **핵심 기능**: 규제 문서 기반 심각도 결정 시스템
- **주요 모듈**:
  - `severity_decision.py`: 규제 문서를 기반으로 Level 1/2/3 결정
  - `xai_explainer.py`: 결정 근거를 설명하는 XAI 모듈
  - `severity_decision_example.py`: 사용 예제
  - `test_severity_decision.py`, `test_xai_explainer.py`: 테스트 파일
- **특징**: 규제 우선 원칙, 결정론적 규칙 기반, 키워드 매칭

### feature/level-router 브랜치
- **핵심 기능**: Level Router 및 E2E 테스트
- **주요 모듈**:
  - `e2e_test.py`: Regulation Agent E2E 테스트 및 Level Router 로직
  - `check_chroma.py`: ChromaDB 체크 스크립트
- **특징**: GuardDuty finding 기반 Level 1/2/3 라우팅, TC1-TC6 테스트 케이스

## 4. 아키텍처 통합 전략

### 4.1 두 시스템의 역할 분리

**Level Router (level_router.py)**
- 목적: GuardDuty finding 기반 빠른 레벨 결정
- 입력: GuardDuty finding, runtime_result
- 출력: LevelDecision (1/2/3)
- 특징: 규제 문서 없이도 빠른 결정 가능

**Severity Decision Engine (severity_decision.py)**
- 목적: 규제 문서 기반 정밀한 심각도 결정
- 입력: SecurityEvent, retrieved_docs (규제 문서)
- 출력: SeverityDecisionResult (Level + 설명)
- 특징: 규제 우선 원칙, 결정론적 규칙

### 4.2 통합 워크플로우

```
1. GuardDuty Finding 발생
   ↓
2. Level Router (level_router.py)
   - Level 1 → Runtime Agent만 실행, 종료
   - Level 2/3 → 다음 단계 진행
   ↓
3. RAG 검색 (test_isms_rag.py)
   - 규제 문서 검색
   ↓
4. Severity Decision Engine (severity_decision.py)
   - 규제 문서 기반 정밀 결정
   ↓
5. XAI Explainer (xai_explainer.py)
   - 결정 근거 설명 생성
```
