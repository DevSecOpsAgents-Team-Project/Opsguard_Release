# 전체 시스템 테스트 가이드

## 📋 개요

이 가이드는 Regulation Agent 시스템의 전체 워크플로우를 테스트하는 방법을 설명합니다.

---

## 🔧 사전 준비

### 1. 환경 변수 설정

`.env` 파일을 생성하고 다음 변수를 설정합니다:

```env
OPENAI_API_KEY=sk-your-api-key-here
CHROMA_PERSIST_DIR=./chroma_db
CHROMA_COLLECTION=csa_ccm_v4
```

**Windows PowerShell:**
```powershell
$env:OPENAI_API_KEY="sk-your-api-key-here"
$env:CHROMA_PERSIST_DIR="./chroma_db"
$env:CHROMA_COLLECTION="csa_ccm_v4"
```

**macOS/Linux:**
```bash
export OPENAI_API_KEY="sk-your-api-key-here"
export CHROMA_PERSIST_DIR="./chroma_db"
export CHROMA_COLLECTION="csa_ccm_v4"
```

### 2. 필수 패키지 설치

```bash
pip install chromadb pydantic openai python-dotenv
```

### 3. ChromaDB 데이터 준비

ChromaDB에 규제 문서가 저장되어 있어야 합니다. 없으면 `test_isms_rag.py`를 실행하여 초기화합니다:

```bash
python test_isms_rag.py
```

각 모듈이 정상 작동하는지 확인합니다.

**RAG 연동 테스트:**

**XAI 연동 테스트:**

### Step 2: E2E (End-to-End) 테스트

전체 워크플로우를 테스트합니다.

#### 2.1 기본 E2E 테스트 실행

```bash
python e2e_test.py
```

이 테스트는 다음 워크플로우를 실행합니다:

```
1. GuardDuty Finding 생성 (모의 데이터)
   ↓
2. Level Router 실행
   - Level 1 → Runtime Agent만 실행, 종료
   - Level 2/3 → 다음 단계 진행
   ↓
3. RAG 검색 (ChromaDB)
   - 규제 문서 검색
   ↓
4. Severity Decision Engine + XAI
   - 규제 문서 기반 정밀 결정
   - 결정 근거 설명 생성
   ↓
5. context_chunks 생성
   - Regulation Agent용 컨텍스트 준비
   ↓
6. Regulation Agent LLM 호출
   - 최종 분석 및 권고사항 생성
   ↓
7. Pydantic 검증
   - 출력 스키마 검증
```

#### 2.2 예상 출력

성공적인 실행 시 다음과 같은 출력을 볼 수 있습니다:

```
=== [Day8] Router Decision ===
selected_level: 2
reasons: ['credential_compromise', 'unusual API calls']

=== [Day7] Query Builder Output ===
query_text: AccessKey 이상행위 credential_compromise unusual API calls
where_filter: {'category': 'IAM'}

=== [Day7] Retrieved Regulations (top-k) ===
- #1 id=IAM-05 title=Access Control category=IAM
...

=== [Severity Decision] 규제 문서 기반 정밀 결정 ===
Assigned Level: 2
Justification: ...
Event Factors: 공개 노출 (Public Exposure), 권한 영향 (Privilege Impact)
Regulatory Signals: 2개
  - IAM-05 (CSA_CCM): 보고 의무 (Reporting Obligation)

=== [Day7] context_chunks (preview) ===
...

=== [Day7] Calling Regulation Agent (LLM) ===
...

=== [Day7] Pydantic Validation ===
✅ Validation passed
```

## 📊 테스트 케이스

### TC-1: AccessKey 이상행위
- **입력**: AccessKey 이상행위 GuardDuty Finding
- **예상**: Level 2
- **특징**: credential_compromise 태그

### TC-2: 낮은 심각도 단발성 정찰
- **입력**: 낮은 심각도 정찰 시도
- **예상**: Level 1 (Router에서 종료)
- **특징**: Level Router에서 즉시 종료

### TC-3: 데이터 유출 신호
- **입력**: 데이터 유출 관련 Finding
- **예상**: Level 3
- **특징**: 데이터 민감도 높음

### TC-4: 권한 상승
- **입력**: 권한 상승 시도
- **예상**: Level 2
- **특징**: privilege_impact = True

### TC-5: 높은 심각도이지만 L3 트리거 없음
- **입력**: 높은 심각도 Finding
- **예상**: Level 2
- **특징**: 규제 문서 부족

### TC-6: L3 신호 존재
- **입력**: L3 규제 신호가 있는 Finding
- **예상**: Level 3
- **특징**: 규제 문서에서 L3 신호 확인

## 🔍 문제 해결

### ChromaDB 연결 실패

**증상:**
```
chromadb.exceptions.InvalidCollectionException
```

**해결:**
1. ChromaDB 디렉토리 확인: `ls ./chroma_db`
2. Collection 이름 확인: `.env` 파일의 `CHROMA_COLLECTION` 값 확인
3. ChromaDB 재초기화: `python test_isms_rag.py`

### OpenAI API 키 오류

**증상:**
```
openai.error.AuthenticationError
```

**해결:**
1. `.env` 파일에 `OPENAI_API_KEY` 설정 확인
2. 환경 변수 확인: `echo $OPENAI_API_KEY` (Linux/Mac) 또는 `echo %OPENAI_API_KEY%` (Windows)
3. API 키 유효성 확인

### Import 오류

**증상:**
```
ModuleNotFoundError: No module named 'xxx'
```

**해결:**
```bash
pip install -r requirements.txt
```

또는 개별 설치:
```bash
pip install chromadb pydantic openai python-dotenv
```
