## 2. 개별 모듈 테스트

### 2.1 Level Router 테스트

**파일**: `level_router.py`

**테스트 방법**:
```python
from level_router import decide_response_level

# TC-1: AccessKey 이상행위 → Level 2
finding = {
    "type": "UnauthorizedAccess:IAMUser/AnomalousBehavior",
    "severity": 5.3,
    "resource": {
        "resourceType": "AccessKey",
        "accessKeyDetails": {"accessKeyId": "AKIA..."}
    }
}
runtime_result = {
    "tags": ["credential_compromise"],
    "key_signals": ["unusual API calls", "external IP"]
}

decision = decide_response_level(finding, runtime_result)
print(f"Level: {decision.selected_level}")
print(f"Reasons: {decision.reasons}")
```

**예상 결과**: `selected_level = 2`

---

### 2.2 Severity Decision Engine 테스트

**파일**: `test_severity_decision.py`

**테스트 방법**:

**1. RAG 연동 테스트 (기본):**
```bash
python test_severity_decision.py
```
- ChromaDB가 설정되어 있어야 합니다
- `.env` 파일에 `CHROMA_PERSIST_DIR`과 `CHROMA_COLLECTION` 설정 필요

**2. 단독 테스트 (모의 데이터):**
```bash
python test_severity_decision.py --standalone
```
- ChromaDB 없이도 실행 가능
- 모의 규제 문서를 사용하여 테스트

**3. XAI 연동 테스트:**
```bash
python test_severity_decision.py --xai
```
- XAI 설명을 포함한 심각도 결정 테스트

**또는 Python 코드로 직접 사용:**
```python
from severity_decision import decide_severity_level, decide_severity_level_with_xai
from test_isms_rag import search_similar_documents
import chromadb
from dotenv import load_dotenv
import os

# ChromaDB 로드
load_dotenv()
chroma_persist_dir = os.getenv("CHROMA_PERSIST_DIR", "./chroma_db")
collection_name = os.getenv("CHROMA_COLLECTION", "csa_ccm_v4")

client = chromadb.PersistentClient(path=chroma_persist_dir)
collection = client.get_or_create_collection(name=collection_name)

# 보안 이벤트
security_event = {
    "event_type": "DataExfiltration",
    "resource_type": "RDSDatabase",
    "exposure": "public",
    "privilege_impact": True,
    "data_sensitivity": "high"
}

# RAG 검색
query = "데이터 유출 침해사고 대응"
retrieved_docs = search_similar_documents(collection, query, top_k=5)

# 심각도 결정 (기본)
result = decide_severity_level(security_event, retrieved_docs)
print(result)

# 심각도 결정 + XAI (설명 포함)
xai_result = decide_severity_level_with_xai(security_event, retrieved_docs)
print(xai_result)
```

**예상 결과**: 
- `assigned_level = 1` (Critical)
- `justification`: 결정 근거 설명
- `triggers`: 이벤트 요인 및 규제 신호

### 2.4 E2E 테스트 (TC1-TC6)

**파일**: `e2e_test.py`

**테스트 방법**:
```bash
python e2e_test.py
```

**테스트 케이스**:
- **TC-1**: AccessKey 이상행위 → Level 2
- **TC-2**: 낮은 심각도 단발성 정찰 → Level 1
- **TC-3**: 데이터 유출 신호 → Level 3
- **TC-4**: 권한 상승 → Level 2
- **TC-5**: 높은 심각도이지만 L3 트리거 없음 → Level 2
- **TC-6**: L3 신호 존재 → Level 3

**예상 결과**: 모든 TC 통과 (✅ TC-1 PASSED 등)

### 2.5 Step 4 LLM 분석 테스트

**파일**: `step4_llm_analysis.py`

```python
        "section": "IAM Access Control",
        "no": 4,

# Step 4 분석
```

```json
    {
      "check_id": "CHK-06",
```

## 6. 성능 테스트

```python
start = time.time()
for _ in range(1000):
    decide_response_level(finding, runtime_result)

print(f"1000 calls: {end - start:.3f}s")
```

## 7. 문제 해결

### 7.1 ChromaDB 연결 오류

**증상**: `Collection not found`

**해결**:
```bash
python load_chroma.py
```

### 7.2 OpenAI API 오류

**증상**: `OPENAI_API_KEY가 설정되지 않았습니다`

**해결**:
- `.env` 파일 확인
- 환경 변수 설정 확인

### 7.3 Import 오류

**증상**: `ModuleNotFoundError: No module named 'level_router'`

**해결**:
- 현재 디렉토리에서 실행 확인
- Python 경로 확인
