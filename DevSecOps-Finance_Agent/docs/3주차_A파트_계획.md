# 3주차 A파트 변경 계획 (최소 변경)

## 목표
- Audit Record 완성 (1회 실행당 1레코드)
- 금지/허용 규칙은 policy 파일로만 관리
- 결과 JSONSchema 검증 옵션 (VALIDATE_OUTPUT_SCHEMA=true)
- 기존 1~2주차 로직 유지

---

## 변경 파일 목록 및 패치 계획

### 커밋 1 (STEP 1): Audit 모듈 추가
- **추가** `src/audit/__init__.py`
- **추가** `src/audit/types.py` — AuditRecord 타입 (dataclass 또는 dict 스펙)
- **추가** `src/audit/builder.py` — `build_audit_record(request, result, context)` → audit dict
- **추가** `src/audit/sink_jsonl.py` — `append_audit_record(record, path)` (append 1 line JSONL)

### 커밋 2 (STEP 3): Constraints 확장 (policy 포맷 호환)
- **수정** `src/recommendation_constraints.py` — `evaluate_constraints(context, policy_constraints)` 추가, 반환: `(forbidden_set, forbidden_list)` where list item = `{action_id, constraint_id, reason_code}`. 기존 `rules`(condition/effect/action_ids) 및 신규 `constraints`(when/effect/actions/reason_code) 둘 다 지원.
- **수정** `policy/v1.0.0/recommendation_constraints.json` — `constraint_id`, `when`, `effect`(FORBID|ALLOW_ONLY|DEPRIORITIZE), `actions`, `reason_code` 포함. 기존 rules 유지 또는 마이그레이션.
- **수정** `src/scoring_engine.py` — `evaluate_constraints` 호출해 forbidden 목록(상세) 받고, 기존처럼 forbid set으로 스코어 제외, 반환에 `forbidden_actions` 리스트 추가 (또는 engine에서 한 번 더 호출해 audit용 사용).

### 커밋 3 (STEP 2): 엔진에 Audit 연결
- **수정** `src/engine.py` — result 생성 후 (에러 없을 때만): `audit_record = build_audit_record(...)`, `append_audit_record(audit_record, path)`, then return result. policy_hash 없으면 scores/recommendations 빈 구조로 audit.
- **추가** 환경변수 또는 기본 경로: `AUDIT_LOG_PATH` (기본 `logs/audit.jsonl`)

### 커밋 4 (STEP 4~5): Manifest·스키마
- **수정** `scripts/build_manifest.py` — recommendation_constraints.json 해시 유지 (이미 포함됨).
- **추가** `schemas/audit_record.schema.json` — audit 레코드 JSON Schema.
- **추가** `src/schema_validate.py` 또는 validate.py 확장 — `validate_audit_record(record)`, `VALIDATE_OUTPUT_SCHEMA`일 때 result/audit 검증.

### 커밋 5 (STEP 6): pytest
- **추가** `tests/test_audit_record_fields.py` — finance_run 1회 후 audit 파일에 1줄, 필수 필드 존재 검증.
- **추가** `tests/test_constraints_policy.py` — S1 high → 격리 forbidden, S3 high → 격리 허용.
- **수정** `tests/test_schema_validation.py` — (선택) output schema 검증 케이스.

## Audit 레코드 필수 필드 (명세)
- audit_id, created_at (ISO-8601 UTC), engine_version, policy_version, policy_hash, assumption_hash, request
- scores: impact_breakdown, likelihood, regulatory_component, profile_applied, final_score
- recommendations: top_actions [{action_id, rank, score}], forbidden_actions [{action_id, constraint_id, reason_code}]
- insights: top_drivers, sensitivity, tradeoffs (빈 배열)
