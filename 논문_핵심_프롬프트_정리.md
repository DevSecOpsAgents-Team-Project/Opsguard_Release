## 2. 심각도 레벨 결정 규칙 프롬프트 (Severity Level Decision Rules)

### Level 1 (Critical) 할당 조건

**규칙 1: 긴급 규제 + 공개 노출 + 권한 영향**
```
IF (규제_긴급성 == True) AND (노출_수준 == "public") AND (권한_영향 == True)
THEN 할당_레벨 = Level_1
JUSTIFICATION: "규제 긴급성 + 공개 노출 + 권한 영향"
```

**규칙 2: 고민감도 데이터 + 공개 노출 + 규제 침해 지표**
```
IF (데이터_민감도 == "high") AND (노출_수준 == "public") AND (Level_1_지표 > 0)
THEN 할당_레벨 = Level_1
JUSTIFICATION: "고민감도 데이터 + 공개 노출 + 규제 침해 지표"
```

**규칙 3: 권한 영향 + 관리자 리소스 + 긴급 규제**
```
IF (권한_영향 == True) AND (리소스_타입 CONTAINS "admin") AND (규제_긴급성 == True)
THEN 할당_레벨 = Level_1
JUSTIFICATION: "권한 영향 + 관리자 리소스 + 긴급 규제"
```

**규칙 4: 규제 Level 1 지표 압도적**
```
IF (Level_1_지표 >= 3)
THEN 할당_레벨 = Level_1
JUSTIFICATION: "규제 Level 1 지표 압도적"
```

### Level 3 (Medium/Low) 할당 조건

**규칙 1: 내부 노출 + 낮은 민감도 + 모니터링 규제**
```
IF (노출_수준 == "internal") AND (데이터_민감도 == "low") AND 
   (Level_3_지표 > Level_1_지표) AND (Level_3_지표 > Level_2_지표)
THEN 할당_레벨 = Level_3
JUSTIFICATION: "내부 노출 + 낮은 민감도 + 모니터링 규제"
```

**규칙 2: 권한 영향 없음 + 로깅 규제**
```
IF (권한_영향 == False) AND (Level_3_지표 >= 2) AND (Level_1_지표 == 0)
THEN 할당_레벨 = Level_3
JUSTIFICATION: "권한 영향 없음 + 로깅 규제"
```

### Level 2 (High) 할당 조건 (기본값)

**Fallback 규칙:**
```
IF (Level_1_조건 == False) AND (Level_3_조건 == False)
THEN 할당_레벨 = Level_2
JUSTIFICATION: "완화 조치가 필요하나 즉시 서비스 종료는 불필요"
```

**보수적 할당 규칙:**
```
IF (규제_문서 == NULL) OR (규제_증거_부족 == True)
THEN 할당_레벨 = Level_2
JUSTIFICATION: "규제 문서 증거가 부족하여 보수적으로 Level 2로 설정"
```

## 4. 규제 의도 분류 프롬프트 (Regulatory Intent Classification)

### 의도 추출 로직
```
FOR EACH 규제_문서 IN 검색된_문서:
    즉시_대응_키워드_개수 = COUNT(문서에서 Level_1_키워드 매칭)
    완화_키워드_개수 = COUNT(문서에서 Level_2_키워드 매칭)
    모니터링_키워드_개수 = COUNT(문서에서 Level_3_키워드 매칭)
    
    IF (즉시_대응_키워드_개수 >= 완화_키워드_개수) AND 
       (즉시_대응_키워드_개수 >= 모니터링_키워드_개수):
        의도 = "immediate action"
    ELIF (완화_키워드_개수 >= 모니터링_키워드_개수):
        의도 = "mitigation"
    ELIF (모니터링_키워드_개수 > 0):
        의도 = "monitoring"
    ELSE:
        의도 = "unknown"
```

```310:348:severity_decision.py
                f"신속한 대응이 필요합니다."
            )
        else:
            # 규제 증거가 부족한 경우 보수적 접근
            justification = (
                f"Level 2 (High) 할당: 규제 문서 증거가 불충분하여 보수적으로 Level 2로 설정. "
                f"이벤트 특성({', '.join(level_2_factors)})을 고려하여 추가 조사가 필요합니다."
            )
        
        triggered_factors.extend(level_2_factors)
        if not triggered_factors:
            triggered_factors.append("기본 Level 2 할당")
        
        return SeverityDecisionResult(
            level=SeverityLevel.LEVEL_2,
            justification=justification,
            triggered_factors=list(set(triggered_factors)),
            regulation_references=[doc.get("id", "") for doc in retrieved_docs[:3]]
        )
```

## 7. 논문 작성 시 강조할 핵심 포인트

### 1. 규제 우선 원칙 (Regulation-First Principle)
- 모든 심각도 결정은 규제 문서의 의도를 최우선으로 고려
- 규제 증거가 부족한 경우 보수적 접근 (Level 2 기본값)

### 2. 결정론적 규칙 기반 (Deterministic Rule-Based)
- 모든 결정은 설명 가능한 규칙으로 수행
- 키워드 매칭과 조건부 로직을 통한 투명한 의사결정

### 3. 다층적 의사결정 구조 (Multi-Layer Decision Structure)
- Level 1: 즉시 대응 필요 (Critical)
- Level 2: 완화 조치 필요 (High, 기본값)
- Level 3: 모니터링/로깅만 (Medium/Low)

### 4. XAI 통합 (Explainable AI Integration)
- 결정 로직과 분리된 설명 생성 레이어
- 규제 신호, 이벤트 요인, fallback 조건을 구조화하여 제공

### 5. 규제 문서 RAG 통합 (Regulatory RAG Integration)
- ChromaDB를 통한 규제 문서 벡터 검색
- 검색된 문서에서 키워드 기반 의도 추출
